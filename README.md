# engineering-playbook

마스터봇과 서브봇 운영을 위한 하네스 엔지니어링 플레이북.

## 목적

이 저장소는 멀티 에이전트 운영 체계를 설계하고 관리하기 위한 기준 문서를 담는다.

핵심 목표:
- 마스터봇과 서브봇의 역할을 분리한다.
- 규칙, 제약, 권한, 승인 체계를 일관되게 관리한다.
- 반복 가능한 실행 하네스를 구축한다.
- 프로젝트별로 재사용 가능한 플레이북을 만든다.

## 기본 구조

- `governance/` 운영 원칙, 정책, 위험 기준
- `agents/` 마스터봇 및 서브봇 정의
- `harness/` 실행 템플릿, 체크리스트, 워크플로우
- `playbooks/` 실제 업무 시나리오별 운영 가이드
- `templates/` 재사용 가능한 문서 템플릿

## 우선 작성할 문서

1. 운영 헌장과 정책 문서
2. 마스터봇 정의서
3. 서브봇 3종 정의서
4. 승인 및 에스컬레이션 규칙
5. 시범 프로젝트용 플레이북

## 초기 원칙

- 마스터봇만 사용자와 최종 계약을 맺는다.
- 서브봇은 위임된 범위 내에서만 실행한다.
- 위험 작업은 승인 게이트를 통과해야 한다.
- 모든 중요한 판단은 기록 가능해야 한다.
- 자동화보다 운영 원칙을 먼저 고정한다.
