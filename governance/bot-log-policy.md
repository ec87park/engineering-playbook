# 봇 로그 기록 정책

## 목적

마스터봇과 각 서브봇이 주고받은 작업 요청과 응답을 추적 가능하게 남겨, 운영 흐름을 복기하고 품질을 개선할 수 있도록 로그 기록 정책을 정의한다.

## 기본 원칙

- 중요한 위임과 응답은 기록으로 남긴다.
- 실제 서브봇 세션 호출은 기본적으로 기록한다.
- 요약 로그보다 원문 로그를 우선한다.
- 마스터봇이 요청한 내용은 가능하면 그대로 기록한다.
- 서브봇이 반환한 응답도 가능하면 그대로 기록한다.
- 민감 정보가 명백히 포함된 경우에만 최소한으로 정리하거나 가린다.

## 볼트 저장 구조

봇 간 작업 로그는 Obsidian vault의 아래 경로에 저장한다.

- `31 Bot Logs/Masterbot/`
- `31 Bot Logs/Memorybot/`
- `31 Bot Logs/Watchbot/`
- `31 Bot Logs/Schedulebot/`
- `31 Bot Logs/Taskbot/`
- `31 Bot Logs/Researchbot/`
- `31 Bot Logs/Builderbot/`
- `31 Bot Logs/Reviewbot/`

## 기록 대상

다음 항목은 기록 우선 대상이다.

- 실제 서브봇 세션 호출
- 역할이 분명한 위임 작업
- 연쇄 위임 결과
- 실패, 보류, 에스컬레이션 사례
- 운영 규칙 개선에 참고할 만한 상호작용

## 기록 형식

각 로그는 가능하면 아래 항목을 포함한다.

- 날짜/시각
- 작업 ID 또는 구분 이름
- 호출 주체
- 대상 봇
- 상태
- 마스터봇 요청 원문
- 서브봇 응답 원문
- 후속 판단 또는 후속 작업

## 운영 규칙

- 실제 서브봇 세션을 생성한 경우 로그 기록을 기본으로 한다.
- 마스터봇은 서브봇 호출 직후 요청 원문을 로그에 남긴다.
- 서브봇 응답이 돌아오면 응답 원문을 그대로 이어서 기록한다.
- 같은 유형의 반복 로그라도 원문 보존을 우선한다.
- 요약은 필요 시 후속 판단 섹션에 별도로 추가할 수 있으나, 원문을 대체하지 않는다.

## 로그 작성 예시

- 마스터봇 요청 원문: 서브봇에 실제로 전달한 호출 메시지 전체
- 서브봇 응답 원문: 서브봇이 반환한 구조화된 결과 전체
- 후속 판단: 이 결과를 이후 어떤 문서화나 다음 단계에 쓸지

## 운영 결론

서브봇 시스템에서 위임과 응답의 흐름은 중요한 운영 자산이다. 따라서 마스터봇과 서브봇의 주요 상호작용은 볼트의 전용 로그 공간에 원문 중심으로 기록하고, 필요할 때만 그 아래에 후속 판단을 덧붙이는 방식을 기본으로 한다.
