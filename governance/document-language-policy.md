# 문서 언어 정책

## 목적

engineering-playbook의 기본 문서 언어를 한글로 통일하여 사용자가 직접 읽고 수정하기 쉽게 유지한다.

## 기본 원칙

- 기본적으로 모든 운영 문서, 정책 문서, 봇 정의 문서, 플레이북, 템플릿 문서는 한글로 작성한다.
- 영어 용어가 필요할 경우에도 가능한 한 한글 설명을 함께 제공한다.
- 파일명은 상황에 따라 영어를 사용할 수 있으나, 문서 본문은 한글을 기본으로 한다.
- 사용자가 직접 검토하고 수정할 수 있도록 문장은 명확하고 과도하게 추상적이지 않게 작성한다.

## 운영 원칙

- 새 문서를 만들 때는 기본적으로 한글로 작성한다.
- 기존 영문 문서는 필요 시 순차적으로 한글화한다.
- 사용자가 별도로 요청한 경우에만 영어 또는 혼합 문서 형식을 사용한다.

## 결론

앞으로 engineering-playbook의 기본 문서 언어는 한글로 유지한다.
