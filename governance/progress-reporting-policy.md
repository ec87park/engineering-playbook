# 진행 보고 정책

## 목적

작업 진행 중 사용자에게 현재 전체 계획에서 무엇이 끝났고 무엇이 남았는지 항상 짧고 명확하게 보여주기 위한 기준을 정의한다.

## 기본 원칙

- 완료 보고 시 현재 작업의 완료 여부만 말하지 않고, 전체 작업 계획 기준으로 완료된 항목과 남은 항목을 함께 보여준다.
- 진행 보고는 길게 늘어놓지 않고 핵심 단계 이름 중심으로 요약한다.
- 세부 실행 중에도 사용자가 현재 위치를 놓치지 않도록 한다.
- 완료된 항목은 기본적으로 최근 5개만 보여준다.
- 사용자가 따로 요청할 때만 전체 완료 목록을 다시 펼쳐 보여준다.

## 표시 형식

가능하면 아래 형식을 따른다.

- 완료된 것
- 남은 것
- 지금 제안하는 다음 단계

예시:
- 완료된 것: 봇 로그 정책 반영, 세션 체크리스트, 빌더봇 구현, 리뷰봇 구현, 스케줄봇 테스트
- 남은 것: 태스크봇 테스트, 워치봇 테스트
- 다음 단계: 태스크봇 실제 테스트

## 운영 결론

앞으로 단계별 완료 보고에는 전체 계획 중 완료된 것과 남은 것을 함께 짧게 표시한다. 완료된 것은 최근 5개만 기본으로 보여준다.
