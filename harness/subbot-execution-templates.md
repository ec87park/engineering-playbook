# 서브봇 실행 템플릿

## 목적

마스터봇이 서브봇을 실제로 호출할 때 사용할 수 있는 공통 실행 템플릿을 정의한다.

이 문서는 각 서브봇에게 어떤 형식으로 작업을 전달해야 하는지, 어떤 출력 형식을 기대해야 하는지, 그리고 공통으로 포함해야 할 제약을 정리한다.

## 공통 템플릿 구조

모든 서브봇 실행 템플릿은 아래 항목을 기본으로 포함한다.

- 역할
- 작업 목적
- 요청 내용
- 작업 범위
- 입력 문맥
- 기대 출력 형식
- 완료 조건
- 제약 사항
- 에스컬레이션 조건

## 1. 메모리봇 실행 템플릿

### 입력 템플릿

- 역할: 메모리 구조화 및 정리
- 작업 목적: 기억 가치가 있는 내용을 적절한 위치에 기록
- 요청 내용: 무엇을 기록하거나 정리할지
- 작업 범위: Daily, Projects, Areas, Knowledge, Archive 중 대상 범위
- 입력 문맥: 사용자 선호, 기존 노트, 최근 대화 핵심
- 기대 출력 형식: 기록 위치, 기록 요약, 승격 여부
- 완료 조건: 적절한 위치에 기록 완료
- 제약 사항: 무차별 저장 금지, 민감 정보 최소화
- 에스컬레이션 조건: 저장 위치 모호, 장기 기억 여부 불확실

## 2. 워치봇 실행 템플릿

### 입력 템플릿

- 역할: 상태 감시 및 변화 보고
- 작업 목적: 중요한 변화나 이상 징후 감지
- 요청 내용: 감시 대상과 보고 기준
- 작업 범위: 일정, 메일, 상태 점검 등 지정 범위
- 입력 문맥: 마지막 상태, 중요도 기준, 보고 빈도
- 기대 출력 형식: 변화 요약, 중요도, 즉시 보고 여부
- 완료 조건: 감시 결과 정리 및 필요 시 보고
- 제약 사항: 불필요한 반복 알림 금지
- 에스컬레이션 조건: 중요도 판단 불확실, 이상 징후 심각

## 3. 스케줄봇 실행 템플릿

### 입력 템플릿

- 역할: 일정 분석 및 요약
- 작업 목적: 일정 구조, 충돌, 준비 필요성 파악
- 요청 내용: 조회 또는 정리할 기간과 목적
- 작업 범위: 오늘, 내일, 이번 주 등
- 입력 문맥: 일정 관리 기준, 중요 일정 정보
- 기대 출력 형식: 일정 요약, 충돌 여부, 준비 시점, 비는 시간
- 완료 조건: 일정 운영에 필요한 핵심 정보 제공
- 제약 사항: 임의 일정 변경 금지
- 에스컬레이션 조건: 일정 충돌 해소에 사용자 판단 필요

## 4. 태스크봇 실행 템플릿

### 입력 템플릿

- 역할: 태스크 정리 및 우선순위화
- 작업 목적: 해야 할 일을 실행 가능한 단위로 정리
- 요청 내용: 수집, 정리, 우선순위화할 작업 목록 또는 목표
- 작업 범위: 오늘, 이번 주, 특정 프로젝트 등
- 입력 문맥: 현재 일정, 목표, 기존 태스크 목록
- 기대 출력 형식: 우선순위 목록, 상태 구분, 다음 실행 항목
- 완료 조건: 실행 가능한 태스크 구조 제시
- 제약 사항: 비현실적 과적재 금지
- 에스컬레이션 조건: 중요도 충돌, 목표 우선순위 불명확

## 5. 리서치봇 실행 템플릿

### 입력 템플릿

- 역할: 조사 및 비교 분석
- 작업 목적: 의사결정에 필요한 근거 수집
- 요청 내용: 조사 주제와 비교 질문
- 작업 범위: 조사 대상, 비교 옵션, 필요한 깊이
- 입력 문맥: 기존 기준, 사용 목적, 판단 기준
- 기대 출력 형식: 요약, 옵션 비교, 근거, 추천 또는 유보 의견
- 완료 조건: 비교 가능한 조사 결과 제공
- 제약 사항: 근거 없는 단정 금지
- 에스컬레이션 조건: 정보 부족, 비교 기준 불명확

## 6. 빌더봇 실행 템플릿

### 입력 템플릿

- 역할: 산출물 생성 및 수정
- 작업 목적: 문서, 구조, 초안, 설정 등 실제 결과 생성
- 요청 내용: 무엇을 만들거나 수정할지
- 작업 범위: 새 문서, 기존 문서, 템플릿, 구조 초안 등
- 입력 문맥: 기존 문서, 정책, 리서치 결과
- 기대 출력 형식: 산출물 초안, 변경 요약, 영향 범위
- 완료 조건: 검토 가능한 결과물 완성
- 제약 사항: 기준 무시 금지, 임의 확정 금지
- 에스컬레이션 조건: 요구사항 불명확, 기준 충돌

## 7. 리뷰봇 실행 템플릿

### 입력 템플릿

- 역할: 검토 및 품질 확인
- 작업 목적: 누락, 충돌, 리스크 식별
- 요청 내용: 검토할 산출물과 검토 기준
- 작업 범위: 문서, 구조, 초안, 설정 등 대상 범위
- 입력 문맥: 요청 원문, 기존 정책, 관련 산출물
- 기대 출력 형식: 검토 요약, 발견 이슈, 수정 권고, 통과 판단
- 완료 조건: 재작업 또는 통과 판단이 가능한 검토 결과 제공
- 제약 사항: 근거 없는 부정 평가 금지
- 에스컬레이션 조건: 기준 충돌, 검토 범위 불명확

## 공통 출력 템플릿

모든 서브봇은 가능하면 아래 형식으로 응답한다.

- 상태
- 결과 요약
- 상세 결과
- 판단 근거
- 남은 리스크
- 후속 제안

## 운영 결론

좋은 실행 템플릿은 서브봇에게 일을 잘 시키기 위한 최소 구조다. 역할만 정의해두고 호출 형식이 없으면 실제 운영에서 흔들리기 쉽다. 이 문서는 마스터봇이 각 서브봇을 일관된 방식으로 호출하기 위한 기준 템플릿이다.
