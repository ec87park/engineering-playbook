# 서브봇 위임 정책

## 목적

마스터봇이 어떤 조건에서 어떤 서브봇에게 작업을 위임할지, 그리고 언제 직접 처리하거나 에스컬레이션할지를 일관되게 판단할 수 있도록 위임 정책을 정의한다.

## 핵심 원칙

### 1. 마스터봇은 기본적으로 통합과 판단을 맡는다

마스터봇은 모든 작업을 직접 처리하기보다, 적절한 서브봇에게 위임하고 결과를 통합하는 역할을 우선한다.

### 2. 위임은 명확한 목적이 있을 때만 한다

서브봇 호출은 단순 분산이 아니라, 역할 특화가 실제로 도움이 될 때만 사용한다.

### 3. 한 번에 필요한 최소 수의 봇만 호출한다

작업을 불필요하게 여러 봇으로 쪼개지 않는다. 단일 봇으로 충분하면 단일 봇으로 끝낸다.

### 4. 충돌과 불확실성은 마스터봇이 회수한다

서브봇 간 판단 충돌, 요구 모호성, 범위 이탈은 마스터봇이 다시 회수해서 조정한다.

## 1. 직접 처리와 위임의 기준

### 마스터봇이 직접 처리할 수 있는 경우

- 단순 질의응답
- 짧은 설명이나 요약
- 별도 전문 역할이 필요 없는 가벼운 정리
- 기존 문서 상태 확인과 간단한 안내

### 서브봇에 위임해야 하는 경우

- 특정 역할의 전문성이 필요한 경우
- 일정, 태스크, 조사, 제작, 검토 등 명확한 작업 유형이 있는 경우
- 구조화된 결과가 필요한 경우
- 후속 단계에서 다른 봇이 재사용할 산출물이 필요한 경우

## 2. 봇별 위임 기준

### 메모리봇

다음 경우 위임한다.
- 기억할 가치가 있는 사실, 선호, 규칙, 결정이 생김
- Daily에서 장기 기록으로 승격이 필요함
- 기존 메모 구조 정리나 중복 정리가 필요함

### 워치봇

다음 경우 위임한다.
- 주기적 상태 감시가 필요함
- 변화 감지와 중요도 필터링이 필요함
- 일정, 메일, 상태 이상을 조용히 감시해야 함

### 스케줄봇

다음 경우 위임한다.
- 일정 조회, 정리, 충돌 탐지가 필요함
- 오늘/주간 일정 요약이 필요함
- 일정 전 준비 시간과 비는 시간 분석이 필요함

### 태스크봇

다음 경우 위임한다.
- 해야 할 일 수집과 정리가 필요함
- 우선순위 재조정이 필요함
- 큰 목표를 실행 단위로 분해해야 함

### 리서치봇

다음 경우 위임한다.
- 정보 조사, 비교, 근거 수집이 필요함
- 선택지 비교와 판단 지원이 필요함
- 근거 기반 요약이 필요함

### 빌더봇

다음 경우 위임한다.
- 문서 초안, 구조안, 템플릿, 설정 초안 생성이 필요함
- 기존 산출물의 수정과 보완이 필요함
- 실제 결과물을 만들어야 함

### 리뷰봇

다음 경우 위임한다.
- 산출물 누락, 충돌, 리스크 점검이 필요함
- 최종 전달 전 검토가 필요함
- 품질 검토와 수정 포인트 도출이 필요함

## 3. 단일 위임과 연쇄 위임

### 단일 위임

하나의 서브봇만으로 충분한 경우.

예:
- 일정 요약 → 스케줄봇
- 태스크 재정리 → 태스크봇
- 조사 요청 → 리서치봇

### 연쇄 위임

한 봇의 결과가 다음 봇의 입력이 되는 경우.

예:
- 리서치봇 → 빌더봇
- 빌더봇 → 리뷰봇
- 스케줄봇 → 태스크봇

### 대표 연쇄 패턴

- 조사 → 제작 → 검토
- 일정 분석 → 태스크 재배치
- 대화 기록 → 메모리 정리

## 4. 회수 기준

다음 상황에서는 마스터봇이 작업을 회수한다.

- 서브봇 결과가 서로 충돌함
- 요구사항이 중간에 바뀜
- 승인 필요 작업이 나타남
- 결과 신뢰도가 낮음
- 범위를 벗어난 판단이 발생함

## 5. 에스컬레이션 기준

서브봇은 다음 경우 반드시 마스터봇으로 되돌린다.

- 요청이 모호하여 방향이 둘 이상 가능함
- 권한이 부족함
- 다른 봇의 판단과 충돌함
- 사용자 확인이 필요한 결정이 있음
- 위험도가 높아짐

## 6. 위임 최소화 원칙

- 단순한 일을 억지로 다단계 위임하지 않는다.
- 위임이 오히려 느리거나 복잡도를 높이면 마스터봇이 직접 처리한다.
- 서브봇은 전문성과 구조화가 필요할 때만 사용한다.

## 7. 운영 결론

좋은 위임 정책은 많이 나누는 것이 아니라, 적절하게 나누는 것이다. 마스터봇은 직접 처리, 단일 위임, 연쇄 위임 중 가장 단순하고 효과적인 경로를 선택해야 하며, 충돌과 불확실성은 다시 회수해서 통합해야 한다.
