# 실제 세션 생성 기준 체크리스트

## 목적

마스터봇이 언제 실제 서브봇 세션을 생성할지 일관되게 판단할 수 있도록 체크리스트 기준을 정의한다.

## 사용 방식

마스터봇은 서브봇 호출이 필요해 보일 때 아래 항목을 순서대로 확인한다.
체크 결과에 따라 직접 처리, 단기 세션 생성, 유지 세션 검토 중 하나를 선택한다.

## 1. 역할 특화 필요성

- 이 작업은 특정 역할의 전문성이 실제로 필요한가?
- 마스터봇이 직접 처리하면 역할 혼선이 생길 가능성이 큰가?
- 결과를 별도 역할 관점으로 분리하는 것이 품질에 도움이 되는가?

## 2. 산출물 재사용성

- 서브봇 결과가 이후 다른 봇이나 문서 작업의 입력으로 다시 쓰일 가능성이 큰가?
- 조사 결과, 초안, 검토 리포트처럼 독립 산출물 가치가 있는가?

## 3. 위임 구조 필요성

- 단일 위임으로 끝나는가?
- 연쇄 위임의 시작 단계인가?
- 복합 작업에서 한 역할을 분리하는 것이 도움이 되는가?

## 4. 작업 범위와 종료 조건

- 작업 범위가 충분히 명확한가?
- 완료 조건이 분명한가?
- 세션을 생성해도 작업 종료 시점을 판단할 수 있는가?

## 5. 문맥 분리 필요성

- 현재 마스터봇 문맥과 분리된 작업 공간이 필요한가?
- 별도 세션으로 분리하면 컨텍스트 오염을 줄일 수 있는가?
- 긴 작업 또는 별도 사고 흐름이 필요한가?

## 6. 비용 대비 효과

- 새 세션을 여는 비용보다 분리 이점이 큰가?
- 이 작업은 굳이 세션까지 만들지 않아도 충분히 처리 가능한가?

## 7. 장기 유지 필요성

- 이 작업은 일회성인가?
- 반복 감시나 장기 추적이 필요한가?
- 유지 세션을 둘 충분한 운영 가치가 있는가?

## 판단 기준

### 직접 처리

다음 조건이면 마스터봇이 직접 처리한다.
- 역할 특화 필요성이 낮음
- 산출물 재사용성이 낮음
- 작업 범위가 짧고 단순함
- 세션 분리 이점이 작음

### 단기 세션 생성

다음 조건이면 실제 서브봇 세션을 생성한다.
- 역할 특화 필요성이 높음
- 산출물 재사용성이 있음
- 작업 범위와 완료 조건이 명확함
- 분리 문맥이 품질에 도움됨

### 유지 세션 검토

다음 조건이면 유지 세션을 검토한다.
- 반복 감시 또는 장기 추적 성격이 있음
- 동일 역할이 반복적으로 호출될 가능성이 큼
- 장기 운영 규율과 재앵커링 계획이 있음

## 현재 권장 적용

- 리서치봇: 단기 세션 생성 우선
- 빌더봇: 단기 세션 생성 우선
- 리뷰봇: 단기 세션 생성 우선
- 워치봇: 유지 세션 또는 반복 실행형 검토
- 메모리봇, 스케줄봇, 태스크봇: 현재는 직접 처리 또는 필요 시 단기 세션

## 운영 결론

실제 세션 생성은 "가능하니까 띄우는 것"이 아니라, 역할 특화, 산출물 가치, 문맥 분리 필요성이 충분할 때만 선택해야 한다. 이 체크리스트는 마스터봇이 그 판단을 일관되게 하기 위한 최소 운영 기준이다.
