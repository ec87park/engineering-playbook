# 서브봇 호출 포맷

## 목적

마스터봇이 실제 서브봇 세션을 생성하거나 호출할 때 사용할 공통 호출 포맷을 정의한다.

이 문서는 문서 수준의 실행 스키마를 실제 호출 메시지 수준으로 더 구체화한 것이다.

## 기본 원칙

- 호출 메시지는 짧지만 구조화되어야 한다.
- 서브봇이 바로 작업을 시작할 수 있도록 목적과 완료 조건이 명확해야 한다.
- 긴 대화 전체를 넘기지 않고 핵심 문맥만 전달한다.
- 출력 형식과 에스컬레이션 기준을 호출 시점에 분명히 적어준다.

## 공통 호출 포맷

호출 메시지는 기본적으로 아래 항목을 포함한다.

### 1. 역할
- 이 세션이 어떤 봇으로 동작해야 하는지 명시

### 2. 작업 목적
- 왜 이 작업을 수행하는지

### 3. 요청 내용
- 실제로 해야 할 일

### 4. 작업 범위
- 어디까지 다뤄야 하는지

### 5. 입력 문맥
- 필요한 문서, 기준, 선행 결과

### 6. 기대 출력
- 어떤 형식으로 결과를 반환해야 하는지

### 7. 완료 조건
- 무엇이 충족되면 완료인지

### 8. 제약 사항
- 하지 말아야 할 것, 승인 필요 조건 등

### 9. 에스컬레이션 조건
- 언제 마스터봇으로 다시 올려야 하는지

## 공통 호출 템플릿 예시

```text
[역할]
당신은 리서치봇이다.

[작업 목적]
프로젝트 구조 결정을 위한 비교 근거를 수집한다.

[요청 내용]
하이브리드 서브봇 구현 방식의 장단점을 조사하고 요약하라.

[작업 범위]
구현 복잡도, 운영 비용, 확장성, 유지보수성 기준으로 비교한다.

[입력 문맥]
- 기존 문서: 서브봇 구현 방식
- 관련 기준: 위임 정책, 실행 템플릿

[기대 출력]
- 상태
- 결과 요약
- 비교 항목별 상세 결과
- 남은 리스크
- 후속 제안

[완료 조건]
마스터봇이 바로 다음 문서 작업에 활용할 수 있을 만큼 구조화된 조사 결과를 반환한다.

[제약 사항]
- 근거 없는 단정 금지
- 불확실한 부분은 불확실하다고 표시

[에스컬레이션 조건]
- 비교 기준이 불명확한 경우
- 필요한 문맥이 부족한 경우
```

## 봇별 호출 포맷 차이

### 리서치봇
강조할 항목:
- 비교 기준
- 근거 수준
- 불확실성 표시

### 빌더봇
강조할 항목:
- 산출물 형식
- 기존 문서와의 연결
- 수정 범위 또는 생성 범위

### 리뷰봇
강조할 항목:
- 검토 대상
- 검토 기준
- 통과/재작업 판단 조건

## 호출 시 포함하지 말아야 할 것

- 불필요하게 긴 전체 대화 로그
- 작업과 무관한 주변 문맥
- 이미 문서에 고정된 내용을 반복해서 장황하게 복붙한 것

## 운영 결론

서브봇 호출 포맷은 실제 멀티 에이전트 운영의 출발점이다. 좋은 호출은 서브봇이 혼란 없이 바로 작업에 들어가게 만들고, 나쁜 호출은 역할과 범위를 흐리게 만든다. 따라서 호출 메시지는 짧더라도 목적, 범위, 출력, 제약을 반드시 포함해야 한다.
