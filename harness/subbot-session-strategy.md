# 실제 서브봇 생성 전략

## 목적

마스터봇이 서브봇을 실제로 언제 생성하고, 언제 재사용하며, 언제 종료할지에 대한 운영 전략을 정의한다.

이 문서는 개념적 봇 정의를 실제 세션 운영 방식으로 연결하는 기준이다.

## 핵심 원칙

### 1. 서브봇은 필요할 때만 만든다

모든 봇을 항상 켜두지 않는다. 작업 목적이 분명하고 역할 특화가 필요한 경우에만 생성한다.

### 2. 기본은 단기 실행이다

대부분의 작업은 단발성 또는 짧은 연쇄 작업으로 처리하고, 작업 종료 후 서브봇 세션도 정리하는 것을 기본 전략으로 둔다.

### 3. 반복성이 높을 때만 유지 세션을 쓴다

반복 감시, 장기 추적, 특정 운영 루틴처럼 같은 역할이 계속 이어질 때만 유지 세션을 고려한다.

### 4. 세션보다 문서가 기준이다

장기 연속성이 필요할 때도 세션 기억보다 레포와 Obsidian 기록을 기준 원본으로 본다.

## 1. 생성 방식

### 단기 실행형 생성

다음 경우 단기 실행형으로 생성한다.

- 조사 요청
- 문서 초안 작성
- 일회성 검토
- 특정 일정 요약
- 특정 태스크 재정리

특징:
- 작업 목적이 명확함
- 완료 조건이 분명함
- 작업이 끝나면 세션을 유지할 필요가 적음

### 유지 세션형 생성

다음 경우 유지 세션형을 고려한다.

- 지속 감시가 필요한 워치봇
- 반복적으로 같은 프로젝트 문맥을 다루는 봇
- 장기간 이어지는 보조 업무

특징:
- 반복 호출 비용을 줄일 수 있음
- 대신 컨텍스트 오염 위험 관리 필요
- 정기 요약과 재앵커링 필요

## 2. 봇별 기본 전략

### 메모리봇
- 기본 전략: 단기 실행형
- 필요 시 주기 정리 작업에 반복 호출

### 워치봇
- 기본 전략: 유지 세션형 또는 반복 실행형
- 감시 성격이 강하므로 장기 운영 가능

### 스케줄봇
- 기본 전략: 단기 실행형
- 일정 확인 요청마다 호출

### 태스크봇
- 기본 전략: 단기 실행형
- 주간 정리 같은 반복 작업은 재호출 방식으로 운영

### 리서치봇
- 기본 전략: 단기 실행형
- 조사 주제 단위로 새로 호출

### 빌더봇
- 기본 전략: 단기 실행형
- 산출물 단위로 호출

### 리뷰봇
- 기본 전략: 단기 실행형
- 검토 대상 단위로 호출

## 3. 생성 판단 기준

마스터봇은 아래 질문으로 생성 여부를 판단한다.

- 이 작업에 역할 특화가 실제로 필요한가
- 작업 범위와 종료 조건이 명확한가
- 단일 응답으로 끝낼 수 있는가, 별도 세션이 더 효율적인가
- 결과가 다른 단계의 입력이 되는가
- 장기 유지가 이득인가, 오염 위험이 더 큰가

## 4. 종료 기준

다음 조건이면 서브봇 세션을 종료 또는 정리한다.

- 완료 조건 충족
- 추가 작업 없음
- 결과가 문서에 고정됨
- 장기 유지 이득이 없음
- 컨텍스트 누적으로 품질 저하 위험이 커짐

## 5. 유지 세션 관리 규칙

유지 세션을 쓰는 경우 아래 규칙을 따른다.

- 주기적으로 핵심 상태를 요약한다.
- 중요한 결과는 문서로 외부화한다.
- 일정 수준 이상 길어지면 새 세션으로 교체한다.
- 세션 자체를 기준 원본으로 삼지 않는다.

## 6. 권장 기본 전략

현재 단계의 권장 전략은 다음과 같다.

- 대부분의 서브봇은 단기 실행형
- 워치봇만 예외적으로 유지 세션 또는 반복 감시형 고려
- 메모리봇은 유지 세션보다 반복 정리형이 적합

## 운영 결론

서브봇을 실제로 운영할 때는 많이 띄우는 것보다 적절한 수명주기를 선택하는 것이 중요하다. 기본은 단기 실행, 예외적으로 반복 감시나 장기 추적이 필요한 경우에만 유지 세션을 사용한다. 장기 연속성은 세션이 아니라 기록된 문서에서 확보한다.
