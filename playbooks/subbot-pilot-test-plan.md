# 서브봇 시범 테스트 계획

## 목적

정의된 마스터봇과 서브봇 체계가 실제 요청 흐름에서 잘 작동하는지 작게 검증하기 위한 시범 테스트 계획을 정의한다.

## 테스트 원칙

- 한 번에 복잡한 전체 시스템을 검증하지 않는다.
- 단일 위임, 연쇄 위임, 복합 위임을 작은 예시로 나눠 점검한다.
- 결과 품질보다 먼저 흐름이 의도대로 작동하는지 확인한다.
- 테스트 결과는 이후 운영 기준 수정에 반영한다.

## 1. 테스트 범위

시범 테스트는 아래 세 가지 흐름부터 검증한다.

### 테스트 A. 단일 위임

목표:
- 마스터봇이 단일 서브봇을 적절히 선택하고 결과를 통합하는지 확인

추천 예시:
- "이번 주 일정 정리해줘" → 스케줄봇
- "오늘 해야 할 일 정리해줘" → 태스크봇

### 테스트 B. 연쇄 위임

목표:
- 한 서브봇의 결과가 다음 서브봇 입력으로 잘 이어지는지 확인

추천 예시:
- "프로젝트 운영 문서 초안 만들어줘" → 리서치봇 → 빌더봇 → 리뷰봇

### 테스트 C. 복합 위임

목표:
- 두 개 이상의 서브봇 결과를 마스터봇이 조정하고 통합하는지 확인

추천 예시:
- "이번 주 일정 보고 할 일 재배치해줘" → 스케줄봇 + 태스크봇

## 2. 테스트 항목

각 테스트에서 아래를 확인한다.

- 마스터봇이 올바른 봇을 선택했는가
- 위임 입력이 충분히 구조화되었는가
- 서브봇 출력이 공통 스키마에 맞는가
- 마스터봇이 결과를 잘 재조합했는가
- 불필요한 위임이 없었는가
- 충돌 발생 시 회수 기준이 작동하는가

## 3. 합격 기준

다음 조건을 기본 합격 기준으로 둔다.

- 위임 대상 선택이 타당함
- 입력과 출력 구조가 문서화된 기준과 대체로 일치함
- 최종 결과가 사용자 요청에 맞게 정리됨
- 오류나 모호성이 생겼을 때 에스컬레이션 또는 회수가 이뤄짐

## 4. 테스트 진행 순서

1. 단일 위임 테스트
2. 연쇄 위임 테스트
3. 복합 위임 테스트
4. 결과 회고
5. 문서 기준 수정 여부 결정

## 5. 권장 첫 테스트

가장 먼저 할 테스트는 단순하고 위험이 낮은 단일 위임 테스트다.

권장 첫 시나리오:
- "오늘 해야 할 일 정리해줘"

이유:
- 태스크봇 단독 흐름 검증 가능
- 일정 연동 없이도 1차 동작 확인 가능
- 결과 평가가 쉬움

## 운영 결론

시범 테스트는 이 체계가 이론상 맞는지만 보는 것이 아니라, 실제 요청에서 마스터봇과 서브봇의 역할 분리가 자연스럽게 작동하는지 확인하는 단계다. 가장 작은 단일 위임부터 시작해 연쇄와 복합 위임으로 확장하는 것이 안정적이다.
