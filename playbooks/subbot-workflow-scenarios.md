# 서브봇 워크플로우 시나리오

## 목적

실제 운영에서 마스터봇이 어떤 흐름으로 서브봇을 호출하고 결과를 통합할지 대표 시나리오를 정의한다.

이 문서는 역할 정의, 위임 정책, 실행 템플릿이 실제 요청 흐름에서 어떻게 결합되는지 보여주는 운영 예시다.

## 시나리오 1. 일정 정리 요청

### 사용자 요청 예시
- 이번 주 일정 정리해줘

### 흐름
1. 마스터봇이 요청 목적을 해석한다.
2. 스케줄봇에 이번 주 일정 조회 및 구조화 작업을 위임한다.
3. 스케줄봇이 중요한 일정, 충돌, 비는 시간, 준비 시점을 요약한다.
4. 마스터봇이 결과를 사용자용 응답으로 통합한다.

### 필요 시 확장
- 태스크와 연결이 필요하면 태스크봇 추가 호출
- 장기 기준 반영이 필요하면 메모리봇 호출

## 시나리오 2. 오늘 해야 할 일 정리

### 사용자 요청 예시
- 오늘 해야 할 일 정리해줘

### 흐름
1. 마스터봇이 현재 목표와 요청 범위를 해석한다.
2. 태스크봇에 현재 태스크 정리와 우선순위화를 위임한다.
3. 필요 시 스케줄봇에 오늘 비는 시간과 일정 충돌 여부를 확인시킨다.
4. 태스크봇이 오늘 집중할 작업을 추린다.
5. 마스터봇이 최종적으로 실행 순서를 사용자에게 전달한다.

## 시나리오 3. 프로젝트 기획 문서 작성

### 사용자 요청 예시
- 새 프로젝트 운영 체계 기획해줘

### 흐름
1. 마스터봇이 요청을 기획 작업으로 해석한다.
2. 리서치봇에 개념, 사례, 비교 근거 수집을 위임한다.
3. 빌더봇에 조사 결과를 바탕으로 초안 문서 작성을 위임한다.
4. 리뷰봇에 누락, 충돌, 리스크 검토를 위임한다.
5. 마스터봇이 검토 결과를 반영해 최종안으로 통합한다.

### 대표 연쇄
- 리서치봇 → 빌더봇 → 리뷰봇

## 시나리오 4. 운영 기준 기억 반영

### 사용자 요청 예시
- 앞으로 이 방식으로 기억해둬

### 흐름
1. 마스터봇이 이 요청이 장기 기준인지 임시 메모인지 판단한다.
2. 메모리봇에 적절한 기록 위치 판단과 저장을 위임한다.
3. 메모리봇이 Daily, Areas, Knowledge 중 적절한 위치에 기록한다.
4. 마스터봇이 이후 유사 요청에 이 기준을 반영한다.

## 시나리오 5. 상태 변화 감시

### 사용자 요청 예시
- 중요한 메일이나 일정 가까워지면 알려줘

### 흐름
1. 마스터봇이 감시 대상과 중요도 기준을 정한다.
2. 워치봇에 감시 작업을 위임한다.
3. 워치봇이 변화 여부를 감지하고 중요도 필터링을 수행한다.
4. 즉시 보고할 변화만 마스터봇 또는 사용자에게 전달한다.

## 시나리오 6. 문서 수정 및 검토

### 사용자 요청 예시
- 이 문서 구조 좀 정리하고 검토해줘

### 흐름
1. 마스터봇이 문서 수정 목적을 파악한다.
2. 빌더봇에 구조 정리와 수정 작업을 위임한다.
3. 리뷰봇에 수정 결과 검토를 위임한다.
4. 마스터봇이 변경 요약과 검토 결과를 함께 사용자에게 전달한다.

### 대표 연쇄
- 빌더봇 → 리뷰봇

## 시나리오 7. 일정과 태스크 결합 운영

### 사용자 요청 예시
- 이번 주 일정 보고 할 일도 같이 재배치해줘

### 흐름
1. 마스터봇이 일정과 태스크를 함께 보는 복합 작업으로 해석한다.
2. 스케줄봇에 일정 구조와 비는 시간 분석을 위임한다.
3. 태스크봇에 할 일 우선순위 정리를 위임한다.
4. 두 결과를 비교해 가능한 배치안을 정리한다.
5. 필요 시 충돌이나 우선순위 문제를 마스터봇이 회수해 조정한다.
6. 사용자에게 통합 운영안으로 전달한다.

## 운영 결론

서브봇 시스템은 각 봇이 따로 존재하는 것보다, 실제 요청 흐름에서 어떻게 연결되는지가 더 중요하다. 이 시나리오들은 마스터봇이 직접 처리, 단일 위임, 연쇄 위임, 복합 위임을 어떤 식으로 선택할 수 있는지 보여주는 기본 운영 패턴이다.
